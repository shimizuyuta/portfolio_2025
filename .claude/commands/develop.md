# /develop

やりたいことを受け取り、マネージャー → 実装 → テスト → レビュー → PR 作成まで自動で行うパイプラインコマンド。

## 使い方

```
/develop <やりたいこと>
```

例:
```
/develop ヒーローセクションにアニメーションを追加したい
/develop お問い合わせフォームを追加したい
```

## 実行フロー

以下のエージェントを順番に起動する。各ステップで失敗した場合はそこで停止し、ユーザーに報告する。

### Step 1: マネージャーエージェント（要件ヒアリング）

`manager` エージェントを起動し、ユーザーの要件を整理する。
不明点のヒアリングを行い、GitHub Issue を作成する。

### Step 2: 実装エージェント

`implementer` エージェントを起動し、Issue の内容を実装する。
ブランチ作成 → 実装 → コミット → Push → PR 作成まで行う。

### Step 3: テストエージェント

`tester` エージェントを起動し、lint と build を実行する。
- 通過 → Step 4 に進む
- 失敗 → エラー内容をユーザーに報告して停止

### Step 4: レビューエージェント

`reviewer` エージェントを起動し、PR の差分をレビューする。
- 問題なし → PR URL をユーザーに報告して完了
- 要修正 → 修正内容をユーザーに報告して停止

## 完了メッセージ

```
✅ パイプライン完了

PR: <PR URL>
レビュー: 問題なし

マージの準備ができました。GitHub でレビューしてマージしてください。
```
