# ホームページ（/）設計

## セクション構成

```
Hero → About → お悩み → Service（→/service）→ Works → Knowledge（→/knowledge）→ Contact
```

---

## Hero セクション

### レイアウト
- PC / SP それぞれに専用のスライドショー実装
- フルワイド背景画像（`opacity-30`）+ `bg-black/40` オーバーレイ
- 下部に波形 SVG ディバイダー（アニメーション）

### 画像
- PC: `public/images/hero/pc/pc_tech.png` / `pc_business.png` / `pc_president.png`
- SP: `public/images/hero/sp/sp_tech.png` / `sp_business.png` / `sp_president.png`
- 5秒ごとに自動スライド（水平スライドアニメーション）

### コピー
```
AI・ITを活用したい。
「でも、誰に頼めばいい？」
ホームページ制作・業務効率化・システム開発など
あなたのビジネスを一緒に作り上げる ITパートナー
```

### CTA ボタン
- なし（Hero にはボタンなし。下スクロール誘導のみ）

## About セクション

### レイアウト
- 写真（左）＋ テキスト（右）
- 「続きを読む」でモーダルを展開

### テキスト（短文）
```
大学を卒業後、新規事業支援のコンサル会社にてエンジニアとして、
クラウドファンディングシステムや行政向けサービス、新規SaaSの立ち上げなど多様なプロジェクトを経験。
現在はフリーランスとして、プライム上場企業や大手メディア企業のシステム開発に携わる傍ら、
中小企業向けにAI活用・システム開発・HP制作／保守運用をワンストップで提供。
```

### 写真
- `public/images/profile.png`（円形表示）

### モーダル
- ミッション文に1文字ずつ出現アニメーション
- 文字表示完了後に蛍光マーカーアニメーション
- モーダル表示中はスクロールロック

## お悩みセクション

### 見出し
```
こんなお悩みありませんか？
```

### レイアウト
- 3カラム（カテゴリ別グループ型）
- 各カラム: 画像（上）＋ カテゴリ名 ＋ 悩みリスト（箇条書き）

### カラム構成

| カラム | 悩み |
|--------|------|
| AI・DX活用 | ChatGPT活用方法がわからない / 自動化ツール選定に迷う |
| Webマーケ | 広告費をかけているのに問い合わせが来ない / SNS集客につながらない |
| Web・システム開発 | 見積が数百万円 / サイト更新できていない / エンジニアと話が噛み合わない |

### 画像素材
- `public/images/problem/` 以下の専用画像を使用

## Service セクション

### 役割
ホーム上の導線。詳細は `/service` に委ねる。

### レイアウト
- 横3カラム、カードなし
- 各カラム: アイコン ＋ サービス名 ＋ ベネフィット文

### サービス構成（3項目）
1. **AI・DX活用支援** — 書類作成・データ集計・返信対応の自動化
2. **Webマーケティング** — 集客からリピートまで ROI が見えるマーケ支援
3. **ホームページ制作・アプリ開発** — 適正コストでスピーディに開発

### CTA
- 「詳しく見る」→ `/service`（一旦非表示中）

---

## Works セクション

### 見出し
```
支援実績
```

### レイアウト
- マーキーアニメーション（左スクロール、ループ）
- ホバーで一時停止

### カード構成（5件）

| 業種 | 内容 | 成果 |
|------|------|------|
| 製造業 | 業務自動化（書類処理・データ入力） | 工数削減 40h/月 |
| EC・小売 | Meta広告運用（ターゲット見直し・A/Bテスト） | CPA 50% 改善 |
| 飲食 | 公式LINE構築・運用（フォロー自動化） | リピート率 20% 向上 |
| 士業 | Webサイトリニューアル + SEO | 問い合わせ数 3倍 |
| 物流 | 在庫管理システム開発（スクラッチ） | 作業時間 60% 削減 |

## アニメーション共通仕様

- Framer Motion を使用
- `fadeInUp`: opacity 0→1、y 40→0（duration 0.7s）
- `staggerContainer`: 子要素を 0.12s ずつ遅延
- `staggerItem`: opacity 0→1、y 28→0（duration 0.55s）
- スクロールで再生（`viewport: { once: false }`）
