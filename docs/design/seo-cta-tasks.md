# ブログ → 無料相談 導線構築 タスク管理

ブログ記事を入口として、無料相談（/contact）への導線を作るための実装・運用タスクを管理する。

## 全体ゴール

ブログ記事を読んだユーザーが「この人に相談したい」と感じ、コンタクトフォームに自然につながる流れを作る。
そのために「Googleに発見してもらえるSEO基盤」「読後に行動を促すCTA」「効果を測る計測基盤」の3つを整える。

---

## フェーズ 1: SEO基盤整備

**目的:** 記事がGoogleにインデックスされ、検索結果に表示されるようにする。
現状は `layout.tsx` が `"use client"` のため Next.js の metadata API が機能しておらず、全ページに `<title>` / `<meta name="description">` がない。ここを直さないと他の施策が無意味になる。

**ゴール:** `/knowledge/[slug]` の各記事ページが Google Search Console でインデックス済みになること。

---

### 1-1. `layout.tsx` を Server Component 化し `Header.tsx` を切り出す

- **目的:** metadata API を有効化するための前提作業。`"use client"` のままではルートレベルの `<title>` タグが機能しない。
- **ゴール:** `layout.tsx` から `"use client"` が消え、ナビゲーションのインタラクション（ドロワー開閉）は `Header.tsx`（Client Component）に移譲されている。
- **対象ファイル:** `src/app/layout.tsx` → `src/app/_components/Header.tsx` を新規作成
- [x] 完了

---

### 1-2. ルートのデフォルト metadata を追加

- **目的:** 全ページ共通の `<title>` テンプレートと、サイト全体のデフォルト description を定義する。
- **ゴール:** `layout.tsx` に `export const metadata` が定義され、`title.template` によって各ページの title が「記事タイトル | YSデベロップメント」形式になっている。
- **対象ファイル:** `src/app/layout.tsx`
- [x] 完了

---

### 1-3. 記事詳細ページに `generateMetadata` を追加

- **目的:** 各記事ページが固有の `<title>` / `<meta name="description">` / OGPタグ / canonical を持つようにする。これが検索順位と SNS シェア品質に直結する。
- **ゴール:** `/knowledge/ai-backoffice-efficiency` などの記事URLをブラウザで開いたとき、`<title>` に記事タイトルが入り、OGPタグが正しく出力されている。
- **対象ファイル:** `src/app/knowledge/[slug]/page.tsx`
- [x] 完了

---

### 1-4. ブログ一覧ページに `metadata` を追加

- **目的:** `/knowledge` ページが適切な title / description を持ち、「ブログ YSデベロップメント」などのクエリで検索結果に表示されるようにする。
- **ゴール:** `/knowledge` ページの `<title>` が「ブログ | YSデベロップメント」になっている。
- **対象ファイル:** `src/app/knowledge/page.tsx`
- [x] 完了

---

### 1-5. `sitemap.ts` を新規作成

- **目的:** Googlebot にサイト内の全URLを通知し、クロール・インデックスを促進する。記事が増えるたびに自動で sitemap に追加されるようにする。
- **ゴール:** `https://www.ysdevelopment.jp/sitemap.xml` にアクセスすると、全公開記事のURLが含まれた XML が返ってくる。
- **対象ファイル:** `src/app/sitemap.ts`（新規作成）
- [x] 完了

---

### 1-6. `robots.ts` を新規作成

- **目的:** クロール可否を明示し、`/admin` や `/api` などをインデックス対象外にする。sitemap のURLも記載して Googlebot に伝える。
- **ゴール:** `https://www.ysdevelopment.jp/robots.txt` に `Allow: /`・`Disallow: /admin/`・`Sitemap:` が含まれている。
- **対象ファイル:** `src/app/robots.ts`（新規作成）
- [ ] 完了

---

### 1-7. Google Search Console にサイト登録・sitemap 送信

- **目的:** Google に「このサイトをクロールしてほしい」と明示的に伝える。sitemap を送信することで全記事URLのインデックスを依頼する。
- **ゴール:** Search Console でサイトが確認済み状態になり、sitemap が「成功」ステータスで登録されている。主要な記事URLを URL検査 ツールでインデックスリクエスト済み。
- **対象:** Google Search Console（運用タスク・コード変更なし）
- [ ] 完了

---

## フェーズ 2: CTA追加（読後の行動促進）

**目的:** 記事を読み終えたユーザーが次のアクション（相談）に進めるように、適切な場所に導線を設置する。
**ゴール:** 記事詳細ページ末尾から `/contact` への遷移が発生し、GA4 でそのクリックが計測されるようになること。

---

### 2-1. 記事末尾に「無料で相談する」CTAセクションを追加

- **目的:** 記事を最後まで読んだユーザー（最も相談意欲が高い層）をコンタクトへ誘導する。
- **ゴール:** 記事本文の直下に sky〜indigo グラデーションの CTA ブロックが表示され、ボタンをクリックすると `/contact` に遷移する。
- **対象ファイル:** `src/app/knowledge/[slug]/page.tsx`（または `_components/ConsultationCta.tsx` として切り出し）
- [ ] 完了

---

### 2-2. 記事末尾に著者プロフィールカードを追加

- **目的:** 「誰が書いたか」を明示することで、相談相手のイメージを持ちやすくし、CTAへの信頼感を高める。
- **ゴール:** CTA の直上に著者名・一言紹介が入ったカードが表示されている。
- **対象ファイル:** `src/app/knowledge/[slug]/page.tsx`
- [ ] 完了

---

### 2-3. 同カテゴリの関連記事3件を表示

- **目的:** 記事を読み終えたユーザーがサイトを離脱するのを防ぎ、別記事への回遊を促す。複数記事を読むことで相談意欲が高まる。
- **ゴール:** CTA の下に同カテゴリ記事3件のカードグリッドが表示される。記事が存在しない場合は非表示。
- **対象ファイル:** `src/lib/knowledge/index.ts`（`getRelatedArticles` 関数追加）+ `src/app/knowledge/[slug]/page.tsx`
- [ ] 完了

---

### 2-4. ブログ一覧ページ最下部に CTAバナーを追加

- **目的:** 一覧ページを閲覧した段階（記事を読む前）のユーザーにも相談の選択肢を提示する。
- **ゴール:** `/knowledge` ページの最下部にダークバックグラウンドの CTAバナーが表示され、「無料で相談する」ボタンが `/contact` に遷移する。
- **対象ファイル:** `src/app/knowledge/page.tsx`
- [ ] 完了

---

### 2-5. コンタクトページの導線テキストを改善

- **目的:** 「お問い合わせ」という汎用的な文言を、「初回無料・24時間以内返信」という具体的なメッセージに変えることで、フォーム送信への心理的ハードルを下げる。
- **ゴール:** `/contact` ページのリード文に「初回相談 無料」「返信は通常24時間以内」「秘密厳守」が明示されている。
- **対象ファイル:** `src/app/contact/page.tsx`
- [ ] 完了

---

## フェーズ 3: 計測基盤

**目的:** 施策の効果を数値で把握し、改善サイクルを回せるようにする。
**ゴール:** GA4 で「記事PV → CTAクリック → フォーム送信」のファネルが追跡できる状態になること。

---

### 3-1. GA4 を導入

- **目的:** サイト全体のアクセス解析基盤を整える。記事ページのPV・流入元・滞在時間を計測する。
- **ゴール:** GA4 のリアルタイムレポートで `/knowledge/[slug]` へのアクセスが計測されている。Vercel に `NEXT_PUBLIC_GA_ID` 環境変数が設定されている。
- **対象ファイル:** `src/app/_components/GoogleAnalytics.tsx`（新規作成）+ `src/app/layout.tsx`
- [ ] 完了

---

### 3-2. CTAクリックをカスタムイベントで計測

- **目的:** 「何人が記事を読んでCTAをクリックしたか」を把握し、CTA のクリック率（CTA clicks / 記事PV）を算出できるようにする。
- **ゴール:** GA4 のイベントレポートに `cta_click` イベントが記録され、`event_label`（記事slug）ごとのクリック数が確認できる。
- **対象ファイル:** `src/app/knowledge/[slug]/_components/ConsultationCta.tsx`
- [ ] 完了

---

### 3-3. Google Search Console と GA4 を連携

- **目的:** 「どの検索クエリからサイトに来たユーザーが相談につながったか」を分析できるようにする。SEOコンテンツ戦略の改善に使う。
- **ゴール:** GA4 の「Search Console リンク」が設定済みで、探索レポートで検索クエリ別のコンバージョン率が確認できる。
- **対象:** Google Search Console + GA4（運用タスク・コード変更なし）
- [ ] 完了

---

## フェーズ 4: コンテンツ戦略（継続）

**目的:** 相談意欲の高いユーザーが検索しそうなキーワードで記事を書き、継続的に流入を増やす。
**ゴール:** 月間オーガニック流入が増加傾向になり、問い合わせ件数が月1件以上になること。

---

### 4-1. ターゲットキーワードで記事を優先的に書く

- **目的:** 「Web制作 費用」「AI活用 中小企業」など、相談意欲と検索ボリュームが高いテーマから記事を執筆する。
- **ゴール:** ティア1キーワードに対応する記事が最低3本公開されている。
- **ティア1（相談意欲×ボリューム高）:** 「Web制作 費用 相場」「AI活用 業務効率化 事例」「フリーランス Web制作 依頼 方法」
- **ティア2（信頼性構築）:** 「Next.js Supabase 個人開発」「Tailwind CSS 入門」
- **対象:** 記事執筆（Supabase 管理画面から投稿）
- [ ] 完了

---

### 4-2. OGP画像を記事ごとに動的生成

- **目的:** SNSシェア時に記事タイトル・カテゴリを含む画像が表示されるようにし、クリック率を高める。
- **ゴール:** X / Facebook でブログ記事URLをシェアすると、記事タイトルが入ったカスタム画像が表示される。
- **対象ファイル:** `src/app/knowledge/[slug]/opengraph-image.tsx`（新規作成、`ImageResponse` を使用）
- [ ] 完了
